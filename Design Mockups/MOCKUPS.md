# Design Mockups

These screenshots represent the visual direction for FitNavi. They
are guides for overall appearance, color palette, spacing, and
component styling — NOT pixel-perfect specifications.

## How to use these mockups

- DO match the overall color scheme, dark background, card styling,
  typography weight, and spacing rhythm visible in these screenshots
- DO use them to understand the intended hierarchy of elements on
  each screen (what's prominent, what's secondary, what's muted)
- DO reference them when styling reusable components
  (FortiFitCard, FortiFitDivider, FortiFitLabel, etc.)
- Do NOT treat specific data, labels, or features shown in the
  mockups as requirements — the PRD.md is the source of truth for
  what to build and what text/data to display
- Do NOT replicate features visible in mockups that are not defined
  in PRD.md — mockups may show earlier designs with features that
  have since been removed or changed

## Priority order

If a mockup conflicts with PRD.md, the PRD always wins.
PRD.md defines WHAT to build. These mockups define HOW it should feel.
